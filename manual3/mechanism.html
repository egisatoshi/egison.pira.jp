Pattern Matching Mechanism

Notations

We need to introduce several notations to explain Egison pattern matching mechanism.
Here is a really brief explanation of each notation.
You can understand the meaning of these notations more deeply, examining the examples in the next section.

Matching State

Egison's pattern matching is reduction of a collection of matching states.
Each matching state has a stack of matching atoms and matching nodes.

Matching Atom

A matching atom is a tuple of a pattern, target, and matcher.

Matching Node

A matching node is similar with a matching state.
Each matching state has a stack of matching atoms and matching nodes.

Examples

Example 1. Non-linear Pattern

Let's examin what will happen when Egison evaluate following pattern matching expressions.

> (test (match-all {2 8 7 2 7} (multiset integer)
          [<cons $m <cons ,m _>> m]))

At first, the initial matching state is generated.
It is as follow.
Matching atoms are represented with a notation like `[p : ... t : ... m : ...]'.
`p' represents pattern.
`t' represents target.
`m' represents matcher.
`env' is the environment when the evalation process enters the `match-all' expression.

MState [stack : {[p : <cons $m <cons ,m _>>
                  t : {2 8 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

The stack of the matching state contains only one matching atom.
This matching atom is reduced with the matcher `(multiset integer)' as specified in the matching atom.
Matching states increases to 5 with this reduction as following.

MState [stack : {[p : $m
                  t : 2
                  m : integer]
                 [p : <cons ,m _>
                  t : {8 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 8
                  m : integer]
                 [p : <cons ,m _>
                  t : {2 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 7
                  m : integer]
                 [p : <cons ,m _>
                  t : {2 8 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 2
                  m : integer]
                 [p : <cons ,m _>
                  t : {2 8 7 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 7
                  m : integer]
                 [p : <cons ,m _>
                  t : {2 8 7 2}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

We focus on the first matching state, for now.
It is following one. 

MState [stack : {[p : $m
                  t : 2
                  m : integer]
                 [p : <cons ,m _>
                  t : {8 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

This matching state is reduced as follow in the next reduction step.
The matcher of the matching atom of the top of the stack is changed to `something' from `integer'.
`something' is the built-in matcher of Egison.
It can match with a wildcard or a pattern variable.

MState [stack : {[p : $m
                  t : 2
                  m : something]
                 [p : <cons ,m _>
                  t : {8 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

This matching state is reduced as follow in the next reduction step.
The matching atom of the top of the stack is popped out, and a new binding is added to the result of pattern matching.

MState [stack : {[p : <cons ,m _>
                  t : {8 7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow in the next reduction step.
The expression in the value pattern `,m' is evaluated with `env' the environment of the matching state, extended with `{[m 2]}' the result of pattern matching in progress.

MState [stack : {[p : ,m
                  t : 8
                  m : integer]
                 [p : _
                  t : {7 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

MState [stack : {[p : ,m
                  t : 7
                  m : integer]
                 [p : _
                  t : {8 2 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

MState [stack : {[p : ,m
                  t : 2
                  m : integer]
                 [p : _
                  t : {8 7 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

MState [stack : {[p : ,m
                  t : 7
                  m : integer]
                 [p : _
                  t : {8 7 2}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

All matching process except third matching state fails and vanishes in the next step.
Third matching state is reduced as follow.
The matching atom of the top of the stack is popped out.

MState [stack : {[p : _
                  t : {8 7 7}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.
The stack is empty now.
This is the end of pattern matching.
The result of pattern matching of this matching state is `{[m 2]}'.

MState [stack : {}
       [env : env]
       [result : {[m 2]}]

The result of other matching states which we ignored at second step is `{[m 7]}'.
Thefore, the result of the whole pattern matching is `{[m 2]}' and `{[m 7]}'.

Example 2. Application of Pattern Function Inside Pattern

(define $twin
  (pattern-function [$pat1 $pat2]
    <cons (& $pat pat1)
     <cons ,pat
      pat2>>))

(test (match-all {1 2 1 3} (multiset integer)
        [<cons $m (twin $n _)> [m n]]))

The initial matching state is as follow.

MState [stack : {[p : <cons $m (twin $n _)>
                  t : {1 2 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

This matching state is reduced as follow.

MState [stack : {[p : $m
                  t : 1
                  m : integer]
                 [p : (twin $n _)
                  t : {2 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 2
                  m : integer]
                 [p : (twin $n _)
                  t : {1 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 1
                  m : integer]
                 [p : (twin $n _)
                  t : {1 2 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

MState [stack : {[p : $m
                  t : 3
                  m : integer]
                 [p : (twin $n _)
                  t : {1 2 1}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

We focused on second matching state, becauase first matching state fails and vanishes.

MState [stack : {[p : $m
                  t : 2
                  m : integer]
                 [p : (twin $n _)
                  t : {1 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

This matching state is reduced as follow.

MState [stack : {[p : $m
                  t : 2
                  m : something]
                 [p : (twin $n _)
                  t : {1 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {}]

This matching state is reduced as follow.

MState [stack : {[p : (twin $n _)
                  t : {1 1 3}
                  m : (multiset integer)]}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.

MState [stack : {(MNode [stack : {[p : <cons (& $pat pat1) <cons ,pat pat2>>
                                   t : {1 1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.

MState [stack : {(MNode [stack : {[p : (& $pat pat1)
                                   t : 1
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

MState [stack : {(MNode [stack : {[p : (& $pat pat1)
                                   t : 1
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

MState [stack : {(MNode [stack : {[p : (& $pat pat1)
                                   t : 3
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 1}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

We focused on first matching state.

MState [stack : {(MNode [stack : {[p : (& $pat pat1)
                                   t : 1
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.
The and pattern is expanded.

MState [stack : {(MNode [stack : {[p : $pat
                                   t : 1
                                   m : integer]
                                  [p : pat1
                                   t : 1
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.
The matcher of the matching atom is changed to `something' from `integer'.

MState [stack : {(MNode [stack : {[p : pat1
                                   t : 1
                                   m : integer]
                                  [p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {[pat 1]}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.
When the pattern of the matching atom of the top of the matching node is a variable, the matching atom of the top of the stack of the matching node is pulled out, and a new matching atom is pushed to the stack of the matching state.
The value of the variable is refer from `penv' of the matching node.

MState [stack : {[p : $n
                  t : 1
                  m : integer]
                 (MNode [stack : {[p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {[pat 1]}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.
The matching atom of the top of the stack is popped out, and a new binding is added to the result of pattern matching.

MState [stack : {[p : $n
                  t : 1
                  m : something]
                 (MNode [stack : {[p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {[pat 1]}])}]
       [env : env]
       [result : {[m 2]}]

This matching state is reduced as follow.

MState [stack : {(MNode [stack : {[p : <cons ,pat pat2>
                                   t : {1 3}
                                   m : (multiset integer)]}]
                        [penv : {[pat1 $n] [pat2 _]}]
                        [env : env1]
                        [result : {[pat 1]}])}]
       [env : env]
       [result : {[m 2] [n 1]}]
